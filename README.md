# 🐾 CatCare AI — ผู้ช่วยดูแลสุขภาพแมว

แอปดูแลสุขภาพแมวส่วนตัว ทำงานแบบ **PWA** (Progressive Web App) เปิดใช้ได้ทั้งบน
เว็บ / Android / iPhone จากไฟล์ชุดเดียว ทำงาน **ออฟไลน์** ได้ และเก็บข้อมูลไว้ในเครื่องของคุณเอง

> ⚠️ **ข้อจำกัดความรับผิดชอบ:** แอปนี้เป็น *ผู้ช่วยดูแลสุขภาพเบื้องต้น* ไม่ใช่ระบบคลินิกหรือโรงพยาบาลสัตว์
> และไม่ทดแทนการวินิจฉัยของสัตวแพทย์ หากแมวมีอาการรุนแรงควรพบสัตวแพทย์ทันที

## 🚀 วิธีใช้งาน

### เปิดใช้ทันที (ง่ายที่สุด)
เปิดไฟล์ `index.html` ด้วยเบราว์เซอร์ได้เลย (ฟีเจอร์ส่วนใหญ่ทำงาน)
> หมายเหตุ: การติดตั้งเป็นแอป (PWA) และ Service Worker ต้องเปิดผ่าน **http/https** ไม่ใช่ `file://`

### เปิดแบบเซิร์ฟเวอร์ (แนะนำ — ได้ครบทุกฟีเจอร์)
```bash
cd catcare
python3 -m http.server 8080
# แล้วเปิด http://localhost:8080 บนเบราว์เซอร์
```

### ติดตั้งลงมือถือ/เดสก์ท็อป (เหมือนแอปจริง)
- **Android (Chrome):** เปิดเว็บ → เมนู ⋮ → "ติดตั้งแอป / Add to Home screen"
- **iPhone (Safari):** เปิดเว็บ → ปุ่มแชร์ → "เพิ่มไปยังหน้าจอโฮม"
- **เดสก์ท็อป (Chrome/Edge):** ไอคอนติดตั้งในแถบ URL

## 🤖 เปิดใช้ AI เต็มรูปแบบ
ไปที่ **เพิ่มเติม → 💾 ข้อมูล → ตั้งค่า AI** ใส่ API key (Claude หรือ OpenAI)
key จะถูกเก็บในเครื่องเท่านั้น ไม่ส่งไปที่ใด นอกจากผู้ให้บริการ AI ที่คุณเลือก

- **Claude:** รองรับทั้งผู้ช่วยตอบคำถามและวิเคราะห์รูปภาพ (vision)
- **OpenAI:** รองรับผู้ช่วยตอบคำถาม

## 🗂️ โครงสร้างไฟล์
```
catcare/
├── index.html      # หน้าแอปหลัก + UI/สไตล์
├── app.js          # ตรรกะทั้งหมด (โปรไฟล์ สมุดสุขภาพ dashboard รายงาน AI backup)
├── symptoms.js     # ฐานความรู้อาการ + เครื่องมือประเมิน (แก้ไข/เพิ่มอาการได้ง่าย)
├── manifest.json   # ข้อมูล PWA (ชื่อ ไอคอน สี)
├── sw.js           # Service Worker (ทำงานออฟไลน์ + cache)
└── icons/          # ไอคอนแอป (192/512)
```

## 💾 ข้อมูลของคุณอยู่ที่ไหน
ทั้งหมดเก็บใน `localStorage` ของเบราว์เซอร์บนเครื่องนี้ ไม่มีการส่งขึ้นเซิร์ฟเวอร์
แนะนำให้กด **ส่งออกไฟล์สำรอง** เป็นระยะ และเมื่อเปลี่ยนเครื่อง/เบราว์เซอร์ให้ใช้ **กู้คืนจากไฟล์**

## 🔧 การต่อยอด (สำหรับผู้พัฒนา)
สถาปัตยกรรมนี้เป็นเวอร์ชัน PWA ที่ใช้งานได้จริง เมื่อพร้อมขยายเป็นระบบเต็ม สามารถต่อยอดเป็น
Next.js + TypeScript (เว็บ), React Native Expo (มือถือ), PostgreSQL + Prisma (ฐานข้อมูล), Docker
โดยยกโครงสร้างข้อมูล (pets / records / logs / assessments / reminders — ทุกตารางมี field `species`)
และฐานความรู้อาการใน `symptoms.js` ไปใช้ต่อได้ทันที
